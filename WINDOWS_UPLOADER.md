# Windows Uploader Guide

> [!WARNING]
> **Disclaimer**: This guide was generated by AI and has not been tested on a Windows machine. Use these instructions at your own risk.

If you are on Windows and cannot use `make`, you can perform the steps manually using PowerShell or Command Prompt.

## Prerequisites
- **OpenSSH Client**: This is usually installed by default on Windows 10 and 11. You can check by running `ssh` in PowerShell.

## 1. Prepare Your Data
1.  Organize the files you want to upload into a folder.
2.  **Zip the files**:
    -   **Right-click** the folder or files -> **Send to** -> **Compressed (zipped) folder**.
    -   **OR** use PowerShell:
        ```powershell
        Compress-Archive -Path "C:\path\to\your\data\*" -DestinationPath "C:\path\to\your\data.zip"
        ```

## 2. Transfer Data
Use the `scp` command to upload your zip file to the server.

### Syntax
```powershell
scp <PATH_TO_ZIP> <REMOTE_USER>@<SERVER_IP>:<REMOTE_PATH>
```

### Example
If your zip file is `data.zip`, your server IP is `1.2.3.4`, your username is `ubuntu`, and you are uploading to the default directory:

```powershell
scp data.zip ubuntu@1.2.3.4:~/private-data-hosting/data/
```

### Shared Directory (Multi-User)
If you are part of a group using a shared directory (e.g., `/srv/shared/`), use that path instead:

```powershell
scp data.zip my_username@1.2.3.4:/srv/shared/
```

## 3. Verify
You can SSH into the server to verify the file arrived:

```powershell
ssh ubuntu@1.2.3.4
ls -l ~/private-data-hosting/data/
```
